### Anatomia visual da lista

```
┌──────────────────────────────┐ ← topo da tela (safe area)
│  [← ] Pokédex                │ header
│  [🔍 Buscar Pokémon...     ] │ search
│  [Tipo ▼]       [A-Z ▼]     │ filtros
├──────────────────────────────┤
│                              │ ← início da lista
│  ┌────────────────────────┐  │
│  │ [img] Bulbasaur   #001 │  │ card 1
│  └────────────────────────┘  │
│  ┌────────────────────────┐  │
│  │ [img] Ivysaur     #002 │  │ card 2
│  └────────────────────────┘  │
│   ...                        │ cards 3..20
│                              │
│     ─ ─ ─ ─ ─ ─ ─ ─ ─      │ ← viewport (borda inferior)
│                              │
│   (sentinel está -200px      │ ← IO dispara aqui, fora da tela
│    abaixo da borda)          │   usuário ainda não chegou
│                              │
│  ┌─────────────────────┐     │
│  │       ◌             │     │ ← spinner (carregando próxima página)
│  └─────────────────────┘     │
│                              │
└──────────────────────────────┘
```

### Estado: carregando próxima página

```
┌──────────────────────────────┐
│  [card 18]                   │
│  [card 19]                   │
│  [card 20]  ← último visível │
│                              │
│       ◌                      │ ← spinner animado (border-t)
│                              │   aparece ANTES do usuário chegar
│  [card 21] ← já renderizado  │   IO foi disparado 200px antes
└──────────────────────────────┘
```

Implementação do spinner (Tailwind):
```html
<div class="flex items-center justify-center py-6" aria-hidden="true">
  <div class="h-5 w-5 animate-spin rounded-full 
              border-2 border-[#cccccc] border-t-[#333333]" />
</div>
```

- `aria-hidden="true"`: leitores de tela não anunciam o spinner
- `border-t` diferente: spinner visual sem dependência de bibliotecas
- `py-6`: espaçamento confortável no fim da lista antes do conteúdo novo

### Estado: fim da lista

```
┌──────────────────────────────┐
│  [card 50]  ← último Pokémon │
│             do filtro ativo  │
│                              │
│   Você viu todos os Pokémon  │ ← mensagem sutil, centralizada
│                              │
│  ╔════════════════════════╗  │
│  ║  Início  Pokédex  ♥   ║  │ ← TabBar
│  ╚════════════════════════╝  │
└──────────────────────────────┘
```

- Sem "X de Y itens" — não é padrão em apps mobile (Instagram, TikTok,
  App Store não mostram contadores na lista)
- Mensagem sutil em texto pequeno (`text-[13px]`, `text-[#999999]`)
- Aparece apenas quando `!hasMore && filtered.length > 0`

### Estado: lista vazia (filtro sem resultados)

```
┌──────────────────────────────┐
│  [🔍 Psyduck             × ] │ ← busca sem resultado
│  [Tipo ▼]       [A-Z ▼]     │
├──────────────────────────────┤
│                              │
│                              │

│                              │   não é responsabilidade do hook
│                              │

```

O hook retorna `visibleItems = []` e `hasMore = false`. O componente
trata o estado vazio com a UI já existente no projeto.

### Comportamento ao trocar filtro

```
ANTES                           DEPOIS (imediato)
─────                           ──────────────────

[card 15] ← usuário estava      topo
[card 16]   aqui                ┌──────────────────┐
[card 17]                       │  [card 1 fogo]   │
[sentinel]                      │  [card 2 fogo]   │
[spinner]                       │  ...             │
                                └──────────────────┘
                                sem animação de scroll
                                behavior: "instant"
```

**Por que `behavior: "instant"` e não `"smooth"`**:
- Animação suave de scroll para o topo levaria ~300ms
- Durante esse tempo o usuário ainda veria os cards do filtro anterior
- Também conflita com o momentum scroll nativo do WebView
- Padrão no mercado: Instagram, TikTok, Google Play → topo instantâneo

### Acessibilidade

**IntersectionObserver e usuários de teclado**:

O IO não requer foco ou interação — detecta visibilidade. Mas usuários
navegando com Tab precisam que o próximo conteúdo já exista no DOM.

Ao navegar por teclado Tab, quando o foco atinge o último card da lista,
o sentinel entra no viewport e o IO carrega mais itens. Isso funciona
**naturalmente** se o `sentinelRef` estiver após o último card.

**Screen readers**:

```html
<!-- O sentinel nunca é anunciado -->
<div ref={sentinelRef} aria-hidden="true" />

<!-- O spinner também não -->
<div aria-hidden="true">
  <div class="animate-spin ..." />
</div>

<!-- A mensagem de fim É anunciada (não tem aria-hidden) -->
<p class="...">Você viu todos os Pokémon</p>
```

**`aria-live` opcional**: se quiser que screen readers anunciem quando
novos itens são carregados, adicione uma region live ao container:

```html
<section aria-live="polite" aria-label="Lista de Pokémon">
  {visibleItems.map(...)}
</section>
```

Atenção: em feeds de scroll infinito, `aria-live` pode ser verboso.
Teste com VoiceOver (iOS) antes de ativar.
